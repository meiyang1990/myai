## 修复 CommentWriter 路径嵌套 bug

- **问题**：`--overwrite` 模式下会在 source_path 目录下创建多余的嵌套子目录
- **根因**：`SourceFile.rel_path` 是相对于 `project_root` 的路径，但 `CommentWriter.output_root` 基于 `source_path` 计算，两者基准不同，`os.path.join(output_root, rel_path)` 导致路径重复嵌套
  - 例：`project_root=/project`，`source_path=/project/src`，`rel_path="src/Main.java"`

- **修复文件**：`comment_writer.py`
- **具体改动**：
  - `write_file()`：overwrite 模式直接使用 `source_file.abs_path` 写回原文件，不再拼接路径
  - `write_file()`：output 模式使用 `os.path.relpath(abs_path, source_path)` 重新计算相对路径，再拼接 `output_root`
  - `__init__()`：overwrite 模式简化 `output_root` 设置，统一指向 `source_path`（仅用于展示）
  - `copy_non_source_files()`：`processed_rels` 从基于 `rel_path`（相对 project_root）改为基于 `abs_path` 转换相对于 `source_path` 的路径，与 `os.walk` 产生的路径基准一致
