# Project Context

## Purpose

**generate_code_comment** 是一个基于 AI 的智能代码注释生成工具。它的核心目标是：

1. **深度理解源码**：自动读取目标项目的源代码文件，分析代码结构、模块关系和业务逻辑
2. **梳理架构设计**：识别并理解项目的整体架构模式、分层设计、模块划分和依赖关系
3. **解析流程设计**：追踪核心业务流程、数据流转路径、调用链路和关键算法逻辑
4. **生成中文注释**：基于对项目的深刻理解，为每个代码文件生成高质量、有意义的中文注释

该工具旨在解决以下痛点：
- 新成员接手项目时理解成本高
- 遗留代码缺少注释，可读性差
- 人工添加注释耗时且容易遗漏关键逻辑
- 注释质量参差不齐，缺乏统一标准

## Tech Stack

- **编程语言**：Python >= 3.11.9
- **AI 框架**：LangChain v0.1.16（langchain-openai v0.1.3）
- **大模型服务**：火山引擎大模型 API（豆包模型，兼容 OpenAI 协议）
- **项目管理**：OpenSpec 变更提案工作流
- **版本控制**：Git

## Project Conventions

### Code Style

- **注释语言**：所有生成的代码注释统一使用简体中文
- **注释风格**：
  - 文件级注释：说明文件的职责、所属模块和核心功能
  - 类级注释：描述类的设计目的、核心职责和关键协作者
  - 方法级注释：解释方法的功能、参数含义、返回值和异常情况
  - 行内注释：针对复杂逻辑、算法和非直观代码添加解释性注释
- **命名规范**：文件和目录使用 kebab-case（短横线连接小写），如 `generate-code-comment`

### Architecture Patterns

- **Skill 驱动**：核心功能以 CodeBuddy Skill 形式实现，封装领域知识和标准化工作流
- **规格驱动开发（OpenSpec）**：所有功能变更通过 proposal → apply → archive 的标准流程管理
- **模块化设计**：
  - 源码读取模块：负责扫描和加载目标项目文件
  - 架构分析模块：识别项目结构、设计模式和模块关系
  - 流程解析模块：追踪业务流程和数据流转
  - 注释生成模块：基于分析结果生成高质量中文注释
  - 注释回写模块：将生成的注释准确插入源码对应位置

### Testing Strategy

- 对生成的注释进行人工审查，确保准确性和可读性
- 验证注释不会破坏原有代码结构和功能
- 覆盖多种编程语言和代码风格的测试用例

### Git Workflow

- 主分支：`main`
- 功能开发：通过 OpenSpec 变更提案流程管理
- 提交信息：使用中文描述，格式为 `[模块] 变更描述`

## Domain Context

本项目属于 **AI 辅助开发工具** 领域，核心知识包括：

- **代码理解（Code Comprehension）**：通过 AST 分析、语义理解等技术深入理解源码含义
- **程序分析（Program Analysis）**：静态分析代码结构、调用关系、数据流等
- **自然语言生成（NLG）**：将代码语义转化为自然语言（中文）描述
- **注释最佳实践**：遵循各语言社区的注释规范（如 Javadoc、JSDoc、Docstring 等）

### 支持的目标项目类型

- Java 项目（Spring Boot、Maven/Gradle）
- 前端项目（React、Vue、TypeScript）
- Python 项目
- Go 项目
- 其他主流编程语言

## Important Constraints

1. **不修改业务逻辑**：工具只添加/更新注释，绝不修改原有代码逻辑
2. **保持代码格式**：注释插入后不破坏原有代码格式和缩进
3. **幂等性**：重复运行不会产生重复注释
4. **安全性**：不将源码内容泄露到外部，所有处理在本地完成
5. **性能约束**：大型项目需支持增量处理，避免一次性加载全部文件

## External Dependencies

- **CodeBuddy IDE**：提供 AI Skill 运行环境和 IDE 集成能力
- **OpenSpec CLI**：管理规格文档和变更提案工作流
- **Git**：版本控制和变更追踪
