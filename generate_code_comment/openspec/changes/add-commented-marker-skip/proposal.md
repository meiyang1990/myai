# Change: 新增"已注释标记"跳过机制与标记回写

## Why

当前系统判断文件是否需要重新处理的唯一手段是进度跟踪器（`progress.json`）。但进度文件与项目源码分离，一旦进度被重置或 `.code_context` 目录被清理，所有文件都会被重新提交给大模型处理，浪费 API 调用和时间。

通过在源码文件头部写入"已注释标记"单行注释，可以让文件本身携带"已处理"的信号。后续即使进度文件丢失，系统也能通过读取源码内容快速识别并跳过已处理文件。

## What Changes

1. **读取源码后检测标记注释**：在生成注释的主循环中，读取到源码文件内容后，检查文件中是否存在包含 `这个文件已经全部加上中文注释` 的单行注释。如果存在该标记，直接跳过该文件并在进度记录中标记为已处理。
2. **注释生成完成后写入标记注释**：当某个文件的注释生成并成功写入后，在输出文件的最开头插入一行对应语言格式的单行注释 `这个文件已经全部加上中文注释`，防止后续重复处理。

## Impact

- Affected specs: `comment-generator`（文件处理进度跟踪、注释回写与格式保持）
- Affected code: `main.py`（do_generate 主循环）、`comment_writer.py`（write_file 方法）、`config.py`（新增标记文本常量和各语言注释格式映射）
