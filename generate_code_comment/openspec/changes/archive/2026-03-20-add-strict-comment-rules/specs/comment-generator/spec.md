## ADDED Requirements

### Requirement: 注释硬性规则

系统 SHALL 在 System Prompt 中定义一组不可违反的注释硬性规则，确保大模型严格遵守注释行为约束，所有规则优先级高于其他注释策略。

#### Scenario: 禁止修改源码

- **WHEN** 大模型处理任意源码文件
- **THEN** 返回的代码逻辑 MUST 与原始代码完全一致
- **AND** 只能添加或修改注释内容，不能改变任何变量名、函数名、逻辑表达式、导入语句或代码结构

#### Scenario: 跳过简单代码行

- **WHEN** 代码行属于以下类型之一：import/from 导入语句、简单变量声明、简单逻辑计算
- **THEN** 不为该代码行生成中文注释

#### Scenario: 简短函数仅添加函数级注释

- **WHEN** 函数体代码在 5 行以内（不含函数定义行和空行）
- **THEN** 仅在函数定义上方添加一条函数级中文注释，说明函数的作用
- **AND** 不为函数体内的各行代码逐行添加注释

#### Scenario: 其他代码逐行简洁注释

- **WHEN** 代码行不属于上述豁免类别（非导入、非简单声明、非简短函数内部）
- **THEN** 为每行代码添加中文注释
- **AND** 注释 MUST 简洁明了，仅说明代码的作用，不做冗长解释

#### Scenario: 注释简洁性要求

- **WHEN** 生成任何中文注释时
- **THEN** 注释 MUST 用最简短的文字说清楚代码的作用
- **AND** 避免使用过于复杂的句式或冗余的修饰词

## MODIFIED Requirements

### Requirement: 多层级注释生成

系统 SHALL 通过 LangChain ChatOpenAI 接入火山引擎大模型，为代码生成文件级、类级、方法级和行内级四个层级的中文注释。系统将整个源码文件连同项目上下文概要一次性提交给大模型，由大模型返回完整的带注释代码。注释生成 MUST 遵守「注释硬性规则」中定义的所有约束。

#### Scenario: 整文件注释生成（带上下文）

- **WHEN** 处理一个源码文件
- **AND** 已有项目上下文概要
- **THEN** 将项目概要作为 System Prompt 的补充上下文
- **AND** 将文件完整内容、文件路径、语言类型信息一起提交给大模型
- **AND** 大模型返回添加了具有项目全局视角的多层级中文注释的完整代码
- **AND** 返回的代码不得修改任何源码逻辑

#### Scenario: 整文件注释生成（无上下文降级）

- **WHEN** 处理一个源码文件
- **AND** 项目上下文概要不可用
- **THEN** 使用现有的无上下文 prompt 方式提交给大模型
- **AND** 行为与变更前一致

#### Scenario: 生成文件级注释

- **WHEN** 大模型处理一个源码文件
- **THEN** 在文件头部生成注释，说明文件所属模块、核心职责和主要功能

#### Scenario: 生成方法级注释

- **WHEN** 大模型处理方法/函数定义
- **THEN** 生成注释说明方法功能、参数含义、返回值、异常情况和核心逻辑流程

#### Scenario: 避免冗余注释

- **WHEN** 代码行为 import/from 导入语句、简单变量声明或简单逻辑计算
- **THEN** 不生成注释
- **AND** 当函数体在 5 行以内时，仅生成函数级注释而非逐行注释
