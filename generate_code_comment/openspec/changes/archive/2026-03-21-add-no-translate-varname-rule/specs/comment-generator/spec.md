## MODIFIED Requirements

### Requirement: 注释硬性规则

系统 SHALL 在 System Prompt 中定义一组不可违反的注释硬性规则，确保大模型严格遵守注释行为约束，所有规则优先级高于其他注释策略。新增第 6 条规则：避免对已有中文注释的代码重复添加注释。新增第 7 条规则：禁止简单翻译变量名/函数名/类名作为注释，注释必须说清代码或函数的实际作用。

#### Scenario: 避免重复添加中文注释

- **WHEN** 代码行或函数体上方已经包含中文注释
- **THEN** 保留已有的中文注释不做修改
- **AND** 不再为该代码行或函数体重复生成新的中文注释
- **AND** 仅对尚无中文注释的代码段落添加新注释

#### Scenario: 禁止翻译变量名式注释

- **WHEN** 为代码行或函数生成注释
- **THEN** 注释 MUST NOT 简单地将变量名、函数名、类名翻译为中文作为注释内容
- **AND** 注释 MUST 说清该段代码或函数的实际作用、业务目的或在整体流程中的角色
- **AND** 例如 `getUserName()` 的注释不应是"获取用户名"，而应说明从哪里获取、用于什么业务场景
- **AND** 例如 `orderList` 的注释不应是"订单列表"，而应说明该列表在当前上下文中承担的角色和数据来源
