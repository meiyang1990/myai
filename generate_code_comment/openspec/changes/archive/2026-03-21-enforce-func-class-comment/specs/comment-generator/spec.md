## MODIFIED Requirements

### Requirement: 注释硬性规则

系统 SHALL 在 System Prompt 中定义一组不可违反的注释硬性规则，确保大模型严格遵守注释行为约束，所有规则优先级高于其他注释策略。新增第 6 条规则：避免对已有中文注释的代码重复添加注释。新增第 7 条规则：禁止简单翻译变量名/函数名/类名作为注释，注释必须说清代码或函数的实际作用。修改第 3 条规则：所有函数/方法无论简单复杂都必须生成函数级注释。新增第 8 条规则：所有类定义无论简单复杂都必须生成类级注释。

#### Scenario: 避免重复添加中文注释

- **WHEN** 代码行或函数体上方已经包含中文注释
- **THEN** 保留已有的中文注释不做修改
- **AND** 不再为该代码行或函数体重复生成新的中文注释
- **AND** 仅对尚无中文注释的代码段落添加新注释

#### Scenario: 禁止翻译变量名式注释

- **WHEN** 为代码行或函数生成注释
- **THEN** 注释 MUST NOT 简单地将变量名、函数名、类名翻译为中文作为注释内容
- **AND** 注释 MUST 说清该段代码或函数的实际作用、业务目的或在整体流程中的角色
- **AND** 例如 `getUserName()` 的注释不应是"获取用户名"，而应说明从哪里获取、用于什么业务场景
- **AND** 例如 `orderList` 的注释不应是"订单列表"，而应说明该列表在当前上下文中承担的角色和数据来源

#### Scenario: 所有函数必须生成函数级注释

- **WHEN** 大模型处理任意函数或方法定义
- **THEN** MUST 在函数/方法定义上方添加函数级中文注释，说明函数的作用、业务目的
- **AND** 无论函数体是简短（5 行以内）还是复杂（超过 5 行），均不得跳过函数级注释
- **AND** 简短函数（5 行以内）不对函数体内部逐行注释，但函数级注释不可省略

#### Scenario: 所有类必须生成类级注释

- **WHEN** 大模型处理任意类定义
- **THEN** MUST 在类定义上方添加类级中文注释，描述类的设计目的、核心职责
- **AND** 无论类是简单（字段少、方法少）还是复杂，均不得跳过类级注释
