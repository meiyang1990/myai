# Change: 强制所有函数和类生成中文注释

## Why

当前 SYSTEM_PROMPT 第 3 条硬性规则规定「简短函数仅添加函数级注释」，但对复杂函数是否需要函数级注释没有明确强制要求（只有第 4 条的「逐行注释」隐含了需要注释）。同时，对于类级注释，当前仅在「注释原则」的「注释层级」中以建议性口吻提及，没有作为硬性规则强制要求。

这导致大模型在实际生成时可能出现：
1. 复杂函数只有行内注释，没有函数级的总体说明注释
2. 简单类（字段少、方法少）被跳过，缺少类级注释
3. 注释质量不一致——有些函数有函数级注释，有些没有

用户明确要求：**无论函数简单还是复杂，都必须生成函数级注释；无论类简单还是复杂，都必须生成类级注释。**

## What Changes

- 修改 `comment_generator.py` 的 `SYSTEM_PROMPT` 第 3 条硬性规则：明确要求所有函数/方法（无论简单复杂）MUST 添加函数级中文注释（简短函数不逐行注释的规则保留）
- 新增第 8 条硬性规则：所有类定义（无论简单复杂）MUST 添加类级中文注释
- 同步修改「注释原则」第 3 条，删除「5 行以内简短函数仅生成函数级注释」的限定表述
- 同步更新 spec：修改「注释硬性规则」Requirement

## Impact

- Affected specs: `comment-generator`（修改「注释硬性规则」Requirement）
- Affected code: `comment_generator.py`（`SYSTEM_PROMPT` 常量）
